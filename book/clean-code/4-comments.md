## Introduction

- Don’t comment bad code — rewrite it
- comment는 필요악임. 표현력이 충분히 좋다면 comment가 필요 없음. comment는 항상 실패임.
- 프로그래머가 comment를 관리를 잘 안해서 오래될수록 구라를 칠 확률이 높음. 코드로 표현해라.
